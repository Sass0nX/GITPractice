- מה עושה הפקודה git init ומה נוצר בעקבותיה? יוצרת repository בכך שנוצרת תיקיית .git
- איפה נשמרים כל האובייקטים של Git אחרי git init? בתיקיית objects כתיקיות נוספות כקובץ hash
- מה ההבדל בין git config --global לבין git config --local? בglobal ההגדרות נשמרות לכל הrepos תחת אותו המשתמש - בlocal לrepo הספציפי
- איך מגדירים שם ואימייל למשתמש ב‑Git ולמה זה חשוב? git config -global user.name maor git config -global user.email "my email" - זה חשוב בשביל מידע שיצביע הcommit על מי ביצע אותו
מה ההבדל בין working directory, staging area ו‑repository? הstaging area זה חלק מהworking flow - כל קובץ שנמצא תחת ניהול של git וביצענו בו שינוי והוא ממתין לcomiit יהיה בשלב זה. - working directory זו התיקייה שאני עובד עליה כרגע - repository זה כל הפרוייקט הספציפי של git
 
- מה זה blob ומה הוא מכיל? blob הוא אובייקט שמכיל את תוכן הקובץ והמציע לקובץ
- מה זה tree ומה הוא מכיל? tree הוא אובייקט שמכיל מציע לכל מבנה הrepo 
- מה זה commit ומה הוא מצביע? הוא אובייקט שמכיל מצביע לtree שהיא מפת התיקייה הנוכחית, מי אישר את הcommit ומי ביצע אותו
- מה זה snapshot ב‑Git? זה תמונה של כל הtree וכל המיגע שנשמר אחרי commit
- למה snapshot לא תופס הרבה מקום בדיסק? כי הוא לא יוצר מידע חדש אם אין צורך בכך- הוא יודע להשתמש במידע ישן blobs ישן וtree ישן אם לא בוצע שינוי.
- מה קורה ל‑blob כשמשנים שם לקובץ? כלום, נשאר זהה
- מה קורה ל‑tree כשמשנים שם לקובץ? נוצר מחדש רק המצביע הספציפי הזה
- מה זה parent commit? הוא הcommit הקודם עליו מבוסס הcommit הנוחכי - עוזר לנו להיסטוריה


תסביר במילים שלך מה branch מייצג ב‑Git. מייצג מצביע נוסף לcommit האחרון - מאפשר להתקדם מאותו commit בלי לשבור את המצביע הנוסף לאותו commit שמבחינתו לא רואה את ההתקדמות כל עוד לא נעשה merge

האם יצירת branch חדש יוצרת tree חדש? התשבוה היא לא - branch חדש הוא מצביע על אותו הcommit האחרון שכאמור כבר מצביע על root tree קיים - בנוסף הפקודה לא מייצרת שום דבר שאמור לגרום לשדינוי הtree


 מתי נוצר קונפליקט ב‑merge - כאשר git מזהה שינוי באותה מקום בדיוק ולא יודע במי לבחור

מה ההבדל בין git diff לבין git merge? שdiff רק עושה בדיקה ולא משנה כלום merge יגרום לשינוי מבציע על של commit .

איך בודקים אם merge ייצור קונפליקט בלי לבצע אותו? git merge --abort

מה ההבדל בין working directory, index, ו‑repository? שrepo הוא המוח של git - מחזיק את כל ההיסטוריה והמידע של הפרוייקט הנוכחי
working dir זה הדיסק הפיזי שמיצג את כל הקבצים בנתיב, Index זה שלב הstaging שם יהיו הקבצים שעברו שינוי ובעצם הsnapshot החדש שנוצר ורק מחכה לאישור
